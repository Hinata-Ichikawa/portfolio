Oritor-site

制作：2017.3

使用したスキル：HTML / CSS / JavaScript

大学新一年生の新歓期間をサポートする二年生のことを、私たちの大学では通称オリターと呼びます。 そして、私たちの大学には一人一つ "芸工ネーム"（芸術工学部のあだ名）があり、一年生の時、先輩の顔も本名も芸工ネームも覚えるのは大変でした。 みんなの顔と名前が載ったサイトがあったらいいのにな！！ってことで Oritor-site(実は制作物はつづりミスしてます。)作りました。

サイトには何を載せようか？

オリターサイトには何を掲載しようかということで、デザインを担当してくれた友達と考えた結果、オリターの説明、新歓期間のスケジュール、オリターのメンバー紹介を載せることに決定。

デザインを作ってもらい、検討、いざ制作開始！

オリターの情報収集

オリター全員の顔写真、芸工ネーム、本名、一言を全員から集めなくてはいけない...。これが一番大変でした。おおよそ全員で90名。学部全員に協力してもらって送ってもらいました。情報を打ち込むのがしんどいったらこれはもう(笑)

このサイトは初めて一人でコーディングしたサイトでした。だから、調べては書き、調べては書き...おかげでだいぶ知識ついた気がします。

そして、初めてのモーダルウィンドウ、横スクロールナビゲーションに結構苦戦しました。

しかし、2Daysで完成せず。

徹夜で制作していたのはよかったものの、翌朝始発の時間に一時解散。2日目の朝、集合時間にいたのは私だけという悲惨な事態に(笑)。このチームはみんな死ぬほど朝に弱かったです...。

途中でメンバーは揃いましたが、私はRailsの知識が皆無だったため、組み込みをすることができず、そのままタイムオーバー。

頑張っても完成しないなら意味がない。うん、よし、『tabitt』作ってやるってこの日に決意しました。

とりあえずRails勉強。そして...

tabitt制作には必須のRuby on Railsをまずは勉強せねば、ということでProgateで徹底的に基礎を叩き込む。

Progateは私のスキルの基礎を固めてくれた良き存在です...。とりあえずRailsのレッスンを2周して、スライドを見ながら制作。初めての言語にエラー連発。

2ヶ月かかりました。でもちゃんと完成しました。
